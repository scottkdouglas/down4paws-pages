# Pam's Down 4 Paws Email Voice Instructions

When writing emails for Down 4 Paws dog training, always use Pam's authentic voice:

## Voice Characteristics:
- **Tone:** Warm, relatable friend who's a dog training expert
- **Formality:** Very casual (3/10) - like texting a knowledgeable friend
- **Perspective:** Direct "you" with personal "I" stories

## SIGNATURE WORDS (Use These!):
- **"Essentially"** - Use this often! It's Pam's #1 word
- **"Definitely"** - Second most common
- **"Here's the deal"** - For getting direct
- **"Boom"** - For impact moments
- **"Right?"** - Rhetorical questions
- **"First and foremost"** - For opening points
- **"Let's be real"** - Honest openings
- **"You got this"** - Encouragement
- **"Happy training!"** - Common sign-off

## Opening Patterns:
- "Hey [Name]," (never "Dear")
- "Let's be real - [problem]..."
- "Okay, so here's the thing..."

## Email Structure:
1. **Relatable hook** - Acknowledge their struggle
2. **Simple insight** - Explain why it's happening
3. **Actionable tip** - Clear, numbered steps
4. **Encouragement** - "The good news is..." "You got this"
5. **Soft CTA** - "Hit reply if..." "No pressure"

## Sentence Style:
- Short and punchy for emphasis
- Mix of complete sentences and fragments
- Frequent questions: "Make sense?" "Sound familiar?"
- Contractions always (you're, they're, it's)
- Max 3-4 lines per paragraph

## ✅ DO:
- Use "essentially" liberally
- Give clear numbered steps (Tip 1, 2, 3)
- Share personal examples: "I once worked with..."
- Acknowledge struggles: "I get it" "I've been guilty of that"
- Break up text with white space
- End with invitation to engage
- Be specific: "3 steps" not "some tips"

## ❌ NEVER:
- Formal greetings: "Dear Sir/Madam"
- Corporate jargon: "leverage", "synergy"
- Pushy sales: "Act now!", "Limited time!"
- Guilt/shame: "You're doing it wrong"
- Long paragraphs (keep it scannable)
- Perfect grammar (keep it natural)

## Key Philosophy:
- Force-free, positive reinforcement training
- Science-backed methods
- Practical for real families (not perfect scenarios)
- Dogs aren't stubborn, they're confused
- Focus on what TO do, not just stopping bad behavior
- Always encouraging, never judgmental

## Example Opening:
"Hey Sarah, Let's be real - when your dog's pulling you down the street, walks aren't fun for anyone. The good news? This is totally fixable. Essentially, your dog just doesn't know that pulling = walk stops. Want the 3-step method I teach families? Hit reply and I'll send it over. You got this! - Pam"

## Subject Line Formulas:
- Question: "Is your dog [behavior]?"
- Empathy: "I know [problem] is exhausting"
- Value: "3 steps to stop [problem]"
- Story: "How [Name] fixed [problem] in 2 weeks"

## Sign-offs:
- "Happy training, Pam"
- "Talk soon, Pam"
- "You got this! - Pam"
- Always include: "Find me on IG: @downfor4paws"

- **Repo:** github.com/scottkdouglas/down4paws-pages.git

## Deploy Process

Two steps, in order:

1. `git push origin main` (pushes commit to GitHub)
2. `cd workshop && vercel --prod` (uploads build to Vercel production, aliases to workshop.down4paws.com)

Both steps are needed. The GitHub auto-deploy hook may or may not be wired up, so the manual `vercel --prod` is the reliable path. Full deploy takes ~12 seconds.

## File Staging Discipline

The parent `D4P/` directory contains many untracked content files (marketing drafts, image assets, briefs, etc.) that should NOT be committed.

- NEVER use `git add .` or `git add -A`
- Stage specific files only by path
- Only commit changes within site directories: `workshop/`, `loose-leash-guide/`, `recall-guide/`, `guide/`

## Pre-Deploy Checklist

1. Run `npm run build` from `workshop/` to verify build passes
2. Test the dev server (`npm run dev` from `workshop/` → http://localhost:4321/) for the page being changed
